🤔 What is an Array?

Definition: An array is a collection of elements of the same data type stored in contiguous memory locations.

🏠 Real-World Analogy

Think of an array as a row of lockers in a school. Each locker has a number (index), they are all the same size, and they are arranged in a straight line.

💾 Memory Analogy

Like hotel rooms in a row — each room has a room number (index 0, 1, 2...), same size, and you can access any room instantly if you know the number.

🎯 Why Use Arrays?

Instead of 100 separate variables (score1, score2...), use scores[100] — organized, efficient, and easy to loop through!

Key Rule: Array indexing starts at 0, not 1! The first element is at index 0.

📢 Array Declaration

Declaration tells the compiler: "I will use an array of this type and size". No memory is allocated yet for the array elements.
// Syntax: data_type array_name[size]; // Declare an array of 5 integers int scores[5]; // Compiler knows: 5 ints coming // Declare an array of 10 floats float prices[10]; // 10 floats, uninitialized // Declare a character array (string) char name[20]; // Can hold 19 chars + null terminator

Rules for Declaration:

  • Size must be a positive integer constant (or constant expression)
  • Size cannot be a variable in standard C89 — use dynamic allocation or VLA (C99)
  • Array name follows the same naming rules as variables
  • Type determines how many bytes each element occupies
Invalid Declarations:
int arr[-5]; → Negative size
int arr[5.5]; → Float size
int arr[]; → Missing size (without initialization)

🏗️ Array Definition & Memory Layout

Definition = Declaration + Memory Allocation. When you write int arr[5];, the compiler allocates 5 x 4 = 20 bytes of contiguous memory.

✨ Array Initialization Methods

Initialization = Giving values to array elements at the time of definition. Always initialize arrays! Uninitialized arrays contain garbage values.

Method 1: All Elements

int arr[5] = {10, 20, 30, 40, 50};

Explicitly set every element. Size can be inferred if omitted.

Method 2: Partial Init

int arr[5] = {10, 20}; // Rest = 0

Remaining elements are automatically set to 0!

Method 3: All Zeros

int arr[5] = {0}; // All = 0

Single zero initializes entire array to zero. Safest method!

Method 4: No Size

int arr[] = {1, 2, 3}; // Size = 3

Compiler counts elements automatically. Size is inferred.

Important: After declaration, you CANNOT initialize the whole array with arr = {1,2,3}; — this only works at definition time! Use a loop or memset() later.

🔤 String Arrays (Character Arrays)

In C, strings are just arrays of characters ending with a special null terminator '\0'.
// Method 1: Character by character (must include null terminator!) char str1[6] = {'H', 'e', 'l', 'l', 'o', '\0'}; // Method 2: String literal (compiler adds \0 automatically!) char str2[6] = "Hello"; // 5 chars + \0 = 6 bytes // Method 3: Size inferred (6 bytes allocated) char str3[] = "Hello"; // Compiler counts: 5 + 1 = 6 // DANGER: Array too small! char str4[5] = "Hello"; // Buffer overflow! Needs 6 bytes!

The null terminator '\0' is essential! Without it, functions like printf("%s") read past the array into garbage memory!

Buffer Overflow Warning: Always allocate size + 1 for strings! char str[5] can only hold 4 characters + '\0'.

📐 2D Arrays (Matrices)

A 2D array is an array of arrays. Think of it as a spreadsheet or a grid with rows and columns.
// Syntax: data_type array_name[rows][cols]; // Declare a 3x3 matrix int matrix[3][3]; // Initialize with values int matrix[3][3] = { {1, 2, 3}, // Row 0 {4, 5, 6}, // Row 1 {7, 8, 9} // Row 2 };

⚠️ Array Rules & Common Pitfalls

🚫 Rule 1: No Bounds Checking

C does NOT check array bounds! Accessing arr[10] when size is 5 reads/writes random memory — undefined behavior!

📏 Rule 2: Fixed Size

Standard C arrays have fixed size at compile time. For dynamic sizes, use malloc() or VLAs (C99).

🔄 Rule 3: No Direct Assignment

You cannot assign one array to another: arr1 = arr2; is ILLEGAL! Use a loop or memcpy().

📍 Rule 4: Array Name = Pointer

The array name is essentially the address of the first element. arr is same as &arr[0].

⚡ Rule 5: sizeof() Trick

sizeof(arr) gives total bytes only in the same scope. In a function parameter, it decays to pointer size!

🧮 Rule 6: Index Starts at 0

Valid indices: 0 to size-1. For arr[5], valid indices are 0,1,2,3,4. Index 5 is OUT OF BOUNDS!

Critical Error Example:
int arr[5]; for(int i=0; i<=5; i++) { // BUG: i=5 is out of bounds! arr[i] = i; // Overwrites memory beyond array! }

🔗 Array vs Pointer Relationship

In C, arrays and pointers are closely related but NOT the same. Understanding this difference is crucial!

Array

int arr[5] = {10, 20, 30, 40, 50}; // arr is the array itself printf("%p", arr); // Address of arr[0] printf("%zu", sizeof(arr)); // 20 bytes (5 x 4) // CANNOT do: // arr = arr2; // ERROR! // arr++; // ERROR!

Pointer

int arr[5] = {10, 20, 30, 40, 50}; int *ptr = arr; // ptr points to arr[0] // ptr is a variable printf("%p", ptr); // Same address as arr printf("%zu", sizeof(ptr)); // 8 bytes (pointer size) // CAN do: ptr++; // Moves to next element printf("%d", *ptr); // 20 (arr[1])
Key Difference: Array name is a constant address (cannot be changed). Pointer is a variable that can point to different addresses. arr[i] is exactly equivalent to *(arr + i).

📤 Passing Arrays to Functions

When you pass an array to a function, C actually passes a pointer to the first element. The array "decays" into a pointer!
// These function declarations are EQUIVALENT: void printArray(int arr[], int size); // Array notation void printArray(int *arr, int size); // Pointer notation (same thing!) // Inside the function: void printArray(int arr[], int size) { printf("%zu", sizeof(arr)); // 8 bytes (pointer size!), NOT array size for(int i=0; i<size; i++) printf("%d ", arr[i]); // Works fine! }
Critical Trap: Inside a function, sizeof(arr) returns the pointer size (8 bytes on 64-bit), NOT the array size! Always pass the size as a separate parameter.

📚 Summary & Key Takeaways

📝 Declaration

int arr[5];
Tells compiler about array. Memory allocated. Contains garbage if not initialized.

✨ Initialization

int arr[5] = {1,2,3,4,5};
Sets values at definition. Partial init fills rest with 0. {0} zeros entire array.

📐 Types

1D, 2D, strings (char arrays). All stored contiguously. 2D uses row-major order. Strings need '\0' terminator.

⚠️ Rules

Index 0 to size-1. No bounds checking. Fixed size. No direct assignment. Array decays to pointer in functions.

💡 Best Practices

Always initialize. Use sizeof(arr)/sizeof(arr[0]) for element count. Pass size to functions. Watch for buffer overflows.

🎯 Remember the Golden Rules

1. Always initialize arrays when you define them.
2. Never access out of bounds.
3. Remember strings need +1 for '\0'.
